Introduction

This guide is for MIEcalculator version 1.0.0, June 2014.

 

In this version there have been some minor changes to the final report format and a Microsoft Word .docx report can now be saved after essentially the same report has been displayed on the screen.  The Word document contains higher resolution graph images and the additional detailed information from the messages box in the Input file list and messages tab of the Report notebook.  This document is intended as a convenient source for copying relevant information into specific report formats.  The print option prints a copy of what is on screen, not the Word document.

 

A minor bug fix now means that setting all transformer errors to zero will not cause a problem provided non-zero uncertainties and coverage factors are given to the zero errors.

 

In other respects the calculator is operating as in version 0.3.3 beta released in September 2013. Changes and issues are all noted in changelog.txt, a file in the Help folder along with a pdf version of this guide.

Getting Started

The calculator installs from a setup.exe file and can be started from either the start menu or the desktop icon (if this option was selected during setup).  As a 32 bit installation its files and data will be found in C:\Program Files on a 32 bit Windows machine or C:\Program Files(x86) on a 64 bit system.   Running MIEcalculator should bring up this window in a few seconds.

 

 

 

 

Next go to File/Open on the menu and navigate to the Projects subfolder that is part of the distribution and open one of the Excel example files. To check that the project has been opened select the Input file list and messages tab and there should be a table filled in with names of various comma separated values (csv) files (note that data in the Excel worksheets will be extracted into .csv files as a background operation).

 

 

 

Now go back to the Certificate Report tab and push the top left hand button labelled Process opened project files.  After 15 seconds or so you should then see a basic Metering Installation Error Report.

 

 

 

Assuming this has worked, you should then check the other tabs.  Note that there is a text box on the Input files lists and messages tab that lists all the calculation parameters in descending order as to how much they each contributed to the overall uncertainty.  The parameter names are a little cryptic, but it is a useful audit tool.  The report itself shows the approximate relative contributions of the meter, CT and VT to the overall uncertainty.  In the File menu the Save item lets you save a Microsoft Word report that incorporates both the screen report and the additional messages.

 

Next go to the Select Notebook menu and have a look at each component in turn, looking at the various tabs.  These worksheets are not protected, so push any buttons at your own peril.  In general the calculator should only throw up error messages rather than crash.  This backroom access is intended for investigations such as trialling different data sets for one component.

 

Creating your own projects

Excel file input

A carefully formatted Excel workbook is used to enter metering data into the calculator and the examples in the project folder can be used as templates.  It is important that the red-tabbed worksheets are kept clean of extraneous data as the individual sheets are used to create the corresponding csv files.  Workbooks can be stored in any directory of your choosing and the calculator will create an xls_temp subdirectory for holding the temporary csv files.  [While access to data in the csv files is useful for debugging purposes, they can be ignored in routine use of the calculator].

 

Please note that the calculator reads the saved version of the workbook and so the workbook must be saved before calculating with MIEcalculator.  Both xls and xlsx formats are accepted.

 

 

Red-tabbed Worksheets

The red-tabbed worksheets are formatted to be read by MIE calculator. It is important that number formatting in Excel does not truncate decimal places in your data as the software may gather the numbers as displayed, rather than their underlying value.

 

The sheet named project holds the master list of csv files that will be created in the background.  Do not alter this list or the names of any of the red-tabbed sheets.  It is safe to insert other sheets for your own data analysis and manipulation.

 

In the example you can see that the InputData sheet organises the basic calibration data and the red-tabbed sheets link back to InputData.  (Feel free to use your own colour scheme for the spreadsheet.)

 

 

Each sheet in turn holds essential data.  For meter the first row gives the number of data points (10) and number of columns (5).  Any number of data points can be accommodated.

 

 

The columns are % of full current, phase angle, error%, uncertainty% and coverage factor.  Additional meter information is in meter_inf, which is a list of influence coefficients.  These all need to be filled in even if the values are zero.  The labels must not be changed.

 

 

Again the first row starts with number of items (7) and number of columns (6).  Column B, D and F are the only numbers that should be changed.  The number of items is fixed for this version of the calculator.

 

The transformers follow a similar pattern, but somewhat tediously there are separate tables for error and phase influence coefficients.  CT has a header row followed by the usual 6 items, 7 columns.  Columns are % of full current, error%, uncertainty%, coverage factor, phase crad, uncertainty crad, coverage factor.  Note that the first row is not used and may be left blank. The examples sometimes have dummy information in that row, but the row is currently ignored.

 

 

 

There is a similar set of worksheets VT, VTe_inf and VTp_inf for the voltage transformer.  Note that for three phase calculations it is appropriate to enter the average error for each point (meter and transformers) but to assign the same uncertainty as for a single phase (i.e. do not divide the uncertainty by √3).

 

Influence variables for the site are given in site_inf.

 

 

Note that the variation in temperature over the year is entered as a mean temperature with an uncertainty as described later under Site temperature.

 

The sheet load contains half hour data for 12 months (ideally), arranged in three columns. The first column has current in %, the second has phase in ° and the third is an active energy measurement.  The calculator sums up all the data and creates a file formatted for plotting a 3D histogram.

 

 

The calculator can be run using the same component information and just changing the load sheet.

 

Note that the spreadsheets can be stored in any directory; they do not need to be kept in the same folder as the examples.

Reporting Errors

When the underlying Python programming language strikes a problem it will pop up a window containing a trace back call.  A copy of this text will tell us exactly where the calculator broke.

 

 

 

Please report problems to keith.jones@callaghaninnovation.govt.nz .

Opening Spreadsheets

Opening a new spreadsheet should clear all text, graphs and temporary files created by the previous project.

Recalculation

After an initial calculation with the Process opened project files button, a recalculation following some changes is possible using the Process manually created files button.   This will use any manually updated data.  It is not recommended to change data points, but it is sometimes useful to change a fitting function.  Pushing Generate Fit on the component data tab and then pushing Generate summary file on the component summary tab will create new values for calculation.  Clearing the previous text and graphs may be necessary.

MIEcalculator Influence Quantities

Introduction

The calculator includes influence quantity data that comes in through various component _inf.csv files. This information is additional to the calibration data, but is potentially the most significant source of uncertainty in the overall error.  All of the listed influence parameters need to be given a value in the calculator and with experience an improved list should evolve.

 

A simple approach has been taken for the calculator and it will be useful to discuss how to improve estimating the effect of all the influence quantities.

Functional Form

At present all influence dependencies are treated as simple linear functions of the form:

temperature error = (temperature  reference temperature) x temperature coefficient

 

or sometimes

temperature error = relative error due to temperature x measured error

 

when the temperature coefficient is expressed in relative terms.

 

The calculator could, in principle, be programmed to use formulae with dependences on e.g. power factor and current, but there is currently insufficient information available to support this approach. 

 

Note that the tables require a single standard deviation estimate and degrees of freedom for each component.

Site temperature

The annual variation in temperature at the site is assumed to have an average temperature of 20 °C with a standard uncertainty of 5 °C, essentially a maximum range of ±15 °C.  No use is currently made of the distribution name, all are assumed to be normal.

Site voltage

Voltage is treated as an influence quantity.  The value is given as % deviation from nominal (nominal being 100% of meter range).  This deviation is assumed to be 0% on average but with a standard deviation of 3%.

Site frequency

Frequency is entered as % deviation from nominal (50 Hz).  Network limits of ±1.5% maximum, or 0.75% standard deviation.

Site harmonics

Both the effect of and level of harmonics is poorly understood, but network limits and IEC specifications give some guidance.  The level has been entered as a distribution from negative to positive % value.

Site CT burden and Site VT burden

The value of the burden is assigned as a property of the site rather than the transformers.  Values are in units of ohm for CTs and units of siemen for VTs and include the uncertainty in the measured values.

Site EM field

It has been decided that EM field issues are temporary disturbances and can be ignored for this uncertainty calculation.  Permanent interference is a design issue that needs to be resolved.  This entry is not being used.

Period of certification

It has been decided that calibration drift with time is unlikely to be known.  Therefore no use is being made of the certification period.  (Only historical calibration data can confirm the choice of drift estimates, but a manufacturers specification might be available.)

Meter temperature coefficient

Set to zero value as most likely, as it could be positive or negative.  Take a maximum allowable value (e.g. for IEC class 0.2) and divide by 2 to give the standard deviation.  No allowance is given for possible dependence on power factor.

Meter calibration temperature

This is set to 23 °C with a standard uncertainty, for example, of 0.1°C representing the variability and accuracy of the laboratory temperature.  The calibration laboratory may have taken the uncertainty in this temperature into account when providing uncertainties in the certificate.  If this is so then the standard deviation can be set to zero.

Meter annual drift

At present no allowance is made for this.

Meter voltage dependence

Again an estimate from IEC maxima (varying by class) can be made.

Meter frequency dependence

Again an estimate from IEC maxima (varying by class) can be made.

Meter field dependence

This is not being included in the uncertainty calculation and so is set to zero.

Meter harmonic dependence

Again an estimate from IEC maxima (varying by class) can be made.

A simplified approach is used here.  The measured error is assumed to have the same temperature coefficient as copper, i.e. it is assumed that winding resistance is a dominant error term.  This couples with a term for the core magnetising impedance.  The variation of error with voltage is then assumed the same as for the same relative variation in frequency.  Burden dependence is linear in 1/burden resistance, and it can be assumed a relative variation in burden will cause the same relative variation in the measured error.

 

For these reasons no explicit coefficient is required for burden or frequency variation.

Fitting Functions

General Comments

The calculator takes calibration data for each component and fits a curve through the calibration points.  Formulas have been selected to give reasonable curves for typical components.  The default is for the transformer functions have three coefficients and the meter function has four coefficients.  There must be more calibration points than the number of coefficients with the difference between the number of points and the number of coefficients giving the degrees of freedom for the fits.  If there are insufficient calibration points then a simpler function must be selected using the drop-down list of functions.

 

A poor fit will lead to an increase in uncertainty, but the error calculation should still succeed.

 

 

Check the Input file list and messages tab to see a report of how well the fitting process went.

Uncertainty Budget

Summary

The Input files list and messages tab in the Overall report notebook carries a summary of the variance contribution of every factor that has been used in the error calculation.  These are arranged in descending order so that the most significant sources of uncertainty can be identified easily.

Files

Installation Folder

The installation directory contains four sub-folders

e_data

All the csv files generated from the data either in the project Excel file or direct from the GUI are created in this folder along with graph images in the form of png files.  You should have no need to interact with the contents of this folder.  If there is a data processing problem, however, the contents of this folder might assist with finding a solution.

Help

This folder contains this htm help file (with images in the UserGuide_files sub folder), a pdf version of this help file and a text file, changelog.txt, that has notes on the history of modifications to the program.   The information in changelog.txt is primarily for the developer, but it should be checked if you have been using an earlier version of MIEcalculator and need to be aware of changes.

mpl-data

This is part of the matplotlib graphing module for Python and should not be modified in any way.

Projects

Example Microsoft Excel spread sheets are provided in this directory.  Once a calculation is completed a report in the form of a Microsoft Word document named, e.g., Example5.xls.docx will be save in this folder.  Note that this Word document will adopt the text styles of the local Word installation, the document only notes the styles: title, header1, header2, paragraph.  The calculation process also triggers the creation of a sub folder xls_temp that contains data extracted from the Excel file.  These are later processed into the e_data folder.

 

If the input Excel file is opened from another folder (the users working directory) then the creation of the Word document and xls_temp sub folder occurs in that working directory.  It is probably preferable to only use the Projects folder for the supplied examples.

Templates

Microsoft Word templates in this folder are used to prepare the Word version of the report.  These should not be deleted.
