1 Introduction

This guide is for MIEcalc3 version 1.2.1, August 2023.

This is the first update to the June 2014 version of the Metering Installation Error Calculator. The main change is that it now reports the error for more than one annual load profile, allowing the performance for typical profiles to be tested along with the actual annual load profile. Other minor changes are to ensure ongoing compatibility with the core programming language, Python 3. Excel workbooks previously used with MIEcalculator 1.0.0 will work with MIEcalc3, but with some changes to the final report format. Improved handling of the uncertainty calculation might show some small changes in the uncertainty compared with the previous software version, but the error value is unchanged.

2 Getting Started

On running MIEcalc3.exe this window should appear within 10 seconds.

If using a Windows installed version, it will state it is running in a Pyinstaller bundle. The file location of the software is always given.

Next go to ‘File/Open workbook’ on the menu and navigate to the ‘Projects’ subfolder that is part of the distribution and open one of the Excel example files.

A brief ‘Reading data … please wait’ message appears while the workbook is being parsed in and the colour of the ‘Process workbook file’ button changes to green to indicate that the data is now ready for processing. Note that the location of temporary data files is given but this is only of interest when debugging or using the ‘Expert/Manual’ process.

Pushing the green ‘Process workbook file’ button now displays some calculation processes in the scrolled text box and finally states ‘Report ready’ in the bottom left of the frame.

Now select the ‘Report’ tab to view the results of the calculations. This short version can be printed directly using ‘File/Quick print’ from the menu. Selecting ‘File/Save word report’ will generate a word report that, in addition to the short report, has all the information from the ‘Main’ tab scrolling text box appended.

This additional information helps with interpreting the quality of the result. The lists all the calculation parameters are given in descending order as to how much they each contributed to the overall uncertainty.  The parameter names are a little cryptic, but it is a useful audit tool.  The report itself shows the approximate relative contributions of the meter, CT and VT to the overall uncertainty.

Next go to the ‘Select Notebook’ menu and have a look at each component in turn, looking at the various tabs within each notebook.  Data in these tabs are not protected, so push any buttons at your own peril.  In general the calculator should only throw up error messages rather than crash.  This backroom access is intended for investigations such as trialling different data sets for one component.

3 Interpreting the Results

MIEcalc3 preselects options for making the error calculations and while these should be suitable for most metering installations it is important to check that there are no obvious problems with the results.

3.1 ‘Main’ tab text box

3.1.1 Goodness of fit

The calculation first carries out function fitting processes to enable interpolation between and limited extrapolation from the measurement data. Messages appear in the text box.

'Good fit.':  No problems. The uncertainty due to the fitting process is consistent with the calibration uncertainty.

‘Fit has lower uncertainty than expected from calibration data, so type B component added.’: This is a common message that simply means that the uncertainty due to the fitting process is negligible, but the calibration uncertainty then needs to be added.

'Data scatter greater than expected, a poor fit.': It is important to look at the relevant graph. It might be that a different fitting function should be used to reduce the uncertainty. The fitting process will have taken account of the poor fit in the uncertainty and so it is safe to still use the fit results, it is just that a lower uncertainty might be possible with a different function.

‘A fit has returned zero uncertainty for a function coefficient. This should only occur if input data has been purposely set to a perfect fit. Message repeats each time the coefficients are used.': This message should only arise when one of the metering components has been set to have zero errors (e.g., no voltage transformer in the installation.

3.1.2 Load messages

The annual load profile is checked when it is loaded into the programme. The following warnings can be given. These messages are likely to be the result of random errors in the annual load profile data. While an error message is given, the calculation proceeds with these points excluded.

'Load error, low or negative current': All currents must be positive (only calculates in two quadrants) and must be greater than 0.1 % of the nominal full range current (errors extrapolated well below 1 % are unreliable).

'Load phase limited to 89.9 degrees maximum.': Avoid calculation failure at +90°.

'Load phase limited to -89.9 degrees minimum.': Avoid calculation failure at -90°.

For each load profile a simple summary of the extreme values is given.

‘Max current = 86.65 %, Min current = 20.35 %, Max phase = 63.10 degrees, Min phase = 2.93 degrees.’

3.1.3 Uncertainty budgets

Check that the most significant components of the uncertainty budget make reasonable sense. See Uncertainty Budget below.

3.2 ‘Total error graph’ tab

This graph should show three surfaces plotted over the range of current and phase angle set by the limits of the relevant annual load profile. The upper surface is the installation error with the expanded uncertainty added and the lower surface is the installation error with the expanded uncertainty subtracted. The middle surface is the error of the metering installation at each combination of current and phase angle.

3.3 ‘Error contours’ tab

This is a quick visual check of the ‘robustness’ of the metering installation. While the installation might be fit for certification with the specific load profile, the contour plot will show if there are areas of load where there might be a risk of failure with a different load profile. This graph is plotted over a wide range of current (1 % to 120 %) and phase angle (-30° to +60°). The maximum absolute value of the error added to the expanded uncertainty is shown. The contour is limited to showing a maximum error of 1.5 % for convenience of display. For installations with very small errors, this contour plot offers little information.

4 Fitting Functions

4.1 General Comments

The calculator takes calibration data for each component and fits a curve through the calibration points.  Formulas have been selected to give reasonable curves for typical components.  The default is for the transformer functions have three coefficients and the meter function has four coefficients.  There must be a greater number of calibration points than the number of coefficients. The difference between the number of points and the number of coefficients gives the degrees of freedom for the fits.  If there are insufficient calibration points, then a function with fewer coefficients must be selected using the drop-down list of functions.

4.2 Changing the fitting function

In the notebooks for each component of the installation there is a ‘xx Data’ tab that includes a choice of fitting function. For example, some meter calibration data

To use a different function, you first reload the file then on the ‘Meter Data’ tab you select a different function from the ‘Select model for curve fit’ list. Then go back to  the ‘Process workbook file’ button and the report is created with the different function.

The new function coefficients are used in the report.

In this case, the reduced chi-square (a measure of the goodness of fit) is smaller for the original tan-ln function than using a simple plane.

5 Influence Quantities

5.1 Introduction

The error of a metering installation depends on the current and phase angle required for the prevailing load and this dependency has been defined by the calibration data. Other quantities are known to influence the error but cannot be directly corrected for and are instead accounted for by calculating the uncertainty in the error. Details of these influence quantities and the sensitivity of the metering installation error to the influence quantities are included in the Excel workbooks. The workbook template will restrict some choices to values derived from IEC standards, but MIEcalc3 will accept any values that are provided. A relatively simple approach is taken for these calculations and while more detailed measurement models are possible, they would need minor modifications to the core code of MIEcalc3. Suggestions for such improvements are welcome.

5.2 Functional Form

At present all influence dependencies are treated as simple linear functions of the form:

temperature error = (temperature – reference temperature) x temperature coefficient

or sometimes

temperature error = relative error due to temperature x measured error

when the temperature coefficient is expressed in relative terms.

For calculation purposes the Excel workbooks need to provide a value, a standard uncertainty, and the degrees of freedom for all the influence quantities and the related sensitivity coefficients.

5.1.1 Site temperature

The annual variation in temperature at the site is assumed to have an average temperature of 20 °C with a standard uncertainty of 5 °C, essentially a maximum range of ±15 °C.  No use is currently made of the distribution name, all are assumed to be normal.

5.1.2 Site voltage

Voltage is treated as an influence quantity.  The value is given as % deviation from nominal (nominal being 100% of meter range).  This deviation is assumed to be 0% on average but with a standard deviation of 3%.

5.1.3 Site frequency

Frequency is entered as % deviation from nominal (50 Hz).  Network limits of ±1.5% maximum, or 0.75% standard deviation.

5.1.4 Site harmonics

Both the effect of and level of harmonics is poorly understood, but network limits and IEC specifications give some guidance.  The level has been entered as a distribution from negative to positive % value.

5.1.5 Site CT burden and Site VT burden

The value of the burden is assigned as a property of the site rather than the transformers.  Values are in units of ohm for CTs and units of siemen for VTs and include the uncertainty in the measured values.

5.1.6 Site EM field

It has been decided that EM field issues are temporary disturbances and can be ignored for this uncertainty calculation.  Permanent interference is a design issue that needs to be resolved.  This entry is not being used.

5.1.7 Period of certification

It has been decided that calibration drift with time is unlikely to be known.  Therefore no use is being made of the certification period.  (Only historical calibration data can confirm the choice of drift estimates, but a manufacturer’s specification might be available.)

Meter temperature coefficient

Set to zero value as most likely, as it could be positive or negative.  Take a maximum allowable value (e.g. for IEC class 0.2) and divide by 2 to give the standard deviation.  No allowance is given for possible dependence on power factor.

Meter calibration temperature

This is set to 23 °C with a standard uncertainty, for example, of 0.1°C representing the variability and accuracy of the laboratory temperature.  The calibration laboratory may have taken the uncertainty in this temperature into account when providing uncertainties in the certificate.  If this is so then the standard deviation can be set to zero.

Meter annual drift

At present no allowance is made for this.

Meter voltage dependence

Again an estimate from IEC maxima (varying by class) can be made.

Meter frequency dependence

Meter field dependence

This is not being included in the uncertainty calculation and so is set to zero.

Meter harmonic dependence

Again, an estimate from IEC maxima (varying by class) can be made.

A simplified approach is used here.  The measured error is assumed to have the same temperature coefficient as copper, i.e. it is assumed that winding resistance is a dominant error term.  This couples with a term for the core magnetising impedance.  The variation of error with voltage is then assumed the same as for the same relative variation in frequency.  Burden dependence is linear in 1/burden resistance, and it can be assumed a relative variation in burden will cause the same relative variation in the measured error.

 

For these reasons no explicit coefficient is required for burden or frequency variation.

6 Files and Folders

6.1 Installation Folder

Folders included in the same directory as MIEcalc3.exe are listed here. For a pyinstaller distribution, there will also be many other folders and files that are essential and must not be deleted.

6.1.1 templates

A Microsoft Word template, default.docx, used to prepare the Word version of the report.  This should not be deleted.

6.1.2 Help

This folder contains this .htm help file (with images in the ‘UserGuide_files’ sub folder). This should not be deleted.

6.1.3 Projects

Example Microsoft Excel spread sheets are provided in this directory.  Once a calculation is completed a report in the form of a Microsoft Word document named, e.g., ‘Example5.xls.docx’ will be save in this folder.  Note that this Word document will adopt the text styles of the local Word installation, the document only notes the styles: title, header1, header2, paragraph.  The calculation process also triggers the creation of a sub folder ‘mie_temp’ that contains data extracted from the Excel file for further processing.

If the input Excel file is opened from another folder (the user’s chosen working directory) then the creation of the Word document and ‘mie_temp’ sub folder occurs in that working directory.  It is probably preferable to only use the ‘Projects’ folder for the supplied examples.

 

Creating your own projects | NOT UPDATED

Excel file input

A carefully formatted Excel workbook is used to enter metering data into the calculator and the examples in the project folder can be used as templates.  It is important that the red-tabbed worksheets are kept clean of extraneous data as the individual sheets are used to create the corresponding csv files.  Workbooks can be stored in any directory of your choosing and the calculator will create an xls_temp subdirectory for holding the temporary csv files.  [While access to data in the csv files is useful for debugging purposes, they can be ignored in routine use of the calculator].

 

Please note that the calculator reads the saved version of the workbook and so the workbook must be saved before calculating with MIEcalculator.  Both xls and xlsx formats are accepted.

 

 

Red-tabbed Worksheets

The red-tabbed worksheets are formatted to be read by MIE calculator. It is important that number formatting in Excel does not truncate decimal places in your data as the software may gather the numbers as displayed, rather than their underlying value.

 

The sheet named ‘project’ holds the master list of csv files that will be created in the background.  Do not alter this list or the names of any of the red-tabbed sheets.  It is safe to insert other sheets for your own data analysis and manipulation.

 

In the example you can see that the ‘InputData’ sheet organises the basic calibration data and the red-tabbed sheets link back to ‘InputData’.  (Feel free to use your own colour scheme for the spreadsheet.)

 

 

Each sheet in turn holds essential data.  For ‘meter’ the first row gives the number of data points (10) and number of columns (5).  Any number of data points can be accommodated.

 

 

The columns are ‘% of full current’, ‘phase angle’, ‘error%’, ‘uncertainty%’ and ‘coverage factor’.  Additional meter information is in ‘meter_inf’, which is a list of influence coefficients.  These all need to be filled in even if the values are zero.  The labels must not be changed.

 

 

Again the first row starts with number of items (7) and number of columns (6).  Column B, D and F are the only numbers that should be changed.  The number of items is fixed for this version of the calculator.

 

The transformers follow a similar pattern, but somewhat tediously there are separate tables for error and phase influence coefficients.  ‘CT’ has a header row followed by the usual 6 items, 7 columns.  Columns are ‘% of full current’, ‘error%’, ‘uncertainty%’, ‘coverage factor’, ‘phase crad’, ‘uncertainty crad’, ‘coverage factor’.  Note that the first row is not used and may be left blank. The examples sometimes have dummy information in that row, but the row is currently ignored.

 

 

 

There is a similar set of worksheets ‘VT’, ‘VTe_inf’ and ‘VTp_inf’ for the voltage transformer.  Note that for three phase calculations it is appropriate to enter the average error for each point (meter and transformers) but to assign the same uncertainty as for a single phase (i.e. do not divide the uncertainty by √3).

 

Influence variables for the site are given in ‘site_inf’.

 

 

Note that the variation in temperature over the year is entered as a mean temperature with an uncertainty as described later under “Site temperature”.

 

The sheet ‘load’ contains half hour data for 12 months (ideally), arranged in three columns. The first column has current in %, the second has phase in ° and the third is an active energy measurement.  The calculator sums up all the data and creates a file formatted for plotting a 3D histogram.

 

 

The calculator can be run using the same component information and just changing the ‘load’ sheet.

 

Note that the spreadsheets can be stored in any directory; they do not need to be kept in the same folder as the examples.

Reporting Software Errors

When the underlying Python programming language strikes a problem a trace-back call will appear in the scrolled text window on the ‘Main’ tab.  A copy of this text will tell us exactly where the calculator struck a problem. It is unusual for the calculator not to continue to respond after a minor error (e.g., loading an incompatible file) and it should continue to work after reloading a new file.

Please report any problems to keith.jones@measurement.govt.nz or tom.stewart@measurement.govt.nz , preferably also providing a copy of the input file that led to the error message.

Opening Spreadsheets

Opening a new spreadsheet will clear all text, graphs and temporary files created by the previous project.

Recalculation

After an initial calculation with the ‘Process workbook file’ button, a recalculation following some changes is possible using the ‘Expert / Manual’ button.   This will use any manually updated data.  This is only recommended for identifying data problems. Pushing ‘Generate Fit’ on the component data tab and then pushing ‘Generate summary file’ on the component summary tab will create new values for calculation.  Clearing the previous text and graphs may be necessary. For choosing different fitting functions for a report, it is recommended that the data is reloaded as explained above in Changing the fitting functions.
